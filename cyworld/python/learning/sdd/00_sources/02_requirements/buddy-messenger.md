# 버디버디(BuddyBuddy) 메신저: 요구사항 원문 (강의 데모용 가상)

> 실재 개인정보·실명·실 서비스 자산 없음. 사용자 식별자는 불투명 예시값(`user-001` 등,
> 데모에서는 시드 닉네임 `yuna`/`minsu`)만 사용. 2000년대 버디버디류 메신저의 핵심
> 개념(버디 리스트, 상태 아이콘, 1:1 대화)을 재해석한 가상 서비스이며 원 서비스의 실제
> UI 자산·상표를 그대로 재현하지 않는다.

- 사용자는 다른 사용자에게 버디(친구) 요청을 보낼 수 있다.
- 상대가 수락하면 양방향 버디 관계가 성립한다.
- 상대가 거절하면 관계가 성립하지 않고, 이후 재요청은 다시 가능하다.
- 이미 버디이거나 대기 중(PENDING) 요청이 있는 대상에게 재요청해도 중복 생성되지 않는다.
- 버디가 아닌 사용자에게는 1:1 메시지를 보낼 수 없다.
- 버디 사이에서는 1:1 채팅 메시지를 주고받을 수 있고, 대화는 시간순으로 양쪽에 동일하게
  보여야 한다.
- 사용자는 온라인/자리비움/오프라인 상태를 표시할 수 있고, 이 상태는 버디에게만 보인다
  (버디가 아닌 사용자에게는 공개되지 않는다).
- 상대가 아직 대화창을 열어보지 않은 메시지는 안읽음으로 남고, 대화를 열람하면 그
  버디가 보낸 메시지가 읽음 처리되어 안읽음 수가 0이 된다.
